# CustDevAI

> Интеллектуальная система автоматизации Customer Development на основе Telegram-чат-бота и нейросетевого анализа русскоязычных текстовых данных.

---

## Что внутри

Программный комплекс автоматизирует полный цикл качественного исследования по методологии Customer Development: проведение глубинных проблемных интервью через Telegram-бота, нейросетевой анализ тональности и тематическое моделирование русскоязычных транскриптов, автоматическую генерацию структурированных аналитических отчётов в формате PDF и XLSX. Архитектурно система декомпозирована на восемь развязанных компонентов с чёткими интерфейсными границами.

Полная спецификация — в каталоге [`docs/`](./docs/), а инструкция для агентского кодирования через Claude Code — в файле [`CLAUDE.md`](./CLAUDE.md).

## Быстрый старт

```bash
# Клонирование и подготовка
git clone <repo-url> custdevai && cd custdevai
cp .env.example .env
# Заполнить .env реальными значениями (см. комментарии в .env.example)

# Запуск стека
docker-compose up --build

# Применение миграций при первом запуске
docker-compose exec api alembic upgrade head

# Доступ:
#   API:     http://localhost:8000  (OpenAPI docs: /api/docs)
#   Web UI:  http://localhost:5173
#   Bot:     запускается автоматически в фоне
```

## Структура репозитория

```
custdevai/
├── CLAUDE.md                        # Инструкция для Claude Code (читается автоматически)
├── README.md                        # Этот файл
├── pyproject.toml                   # Зависимости и конфигурация Python
├── docker-compose.yml               # Локальный запуск полного стека
├── .env.example                     # Шаблон переменных окружения
├── .gitignore                       # Защита от коммита секретов и артефактов
└── docs/                            # Документация ВКР (источник требований)
    ├── 00_thesis_assignment.md      # Задание на ВКР
    ├── 01_theoretical_chapter.md    # Глава 1 — теоретическое обоснование
    ├── 02_analytical_chapter.md     # Глава 2 — анализ предметной области
    └── 03_requirements_specification.md  # Глава 3 — спецификация требований (FR/NFR)
```

По мере разработки появятся каталоги `apps/` (исходный код), `tests/`, `alembic/` (миграции БД), `docker/` (Dockerfile-ы для каждого сервиса) и `scripts/` (вспомогательные утилиты).

## Технологический стек

**Backend.** Python 3.10+, FastAPI, aiogram 3.x, SQLAlchemy 2.x, Celery + Redis, PostgreSQL 14.

**ML.** RuBERT (DeepPavlov) для анализа тональности, BERTopic + multilingual-e5 для тематического моделирования. Все модели исполняются строго локально без передачи данных внешним API.

**Frontend.** React 18 + TypeScript, Vite, TanStack Query.

**Infra.** Docker, Selectel cloud, GitHub Actions CI/CD.

## Лицензия

Проект разрабатывается в рамках ВКР и трудовых отношений с маркетинговым агентством «Отважный маркетинг». Условия использования согласовываются индивидуально.
